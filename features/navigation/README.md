# МОДУЛЬ НАВИГАЦИИ (NAVIGATION FEATURE)

## Ключевая задача:
Реализовать компактный, безопасный и полупрозрачный слой навигации для пешеходов и водителей двухколесного транспорта в Южной Корее на базе Naver Map SDK.

## Инженерные требования (Закон 0% нагрева и 0 вон):
1. ЗАПРЕЩЕНО разворачивать карту во весь экран очков. Навигация должна рендериться в виде компактного 2D HUD-виджета (Picture-in-Picture) в углу обзора (занимать не более 10-15% площади).
2. Слой виджета должен быть полупрозрачным (базово 40-50% с возможностью автонастройки под яркость улицы), чтобы не перекрывать реальную дорогу.
3. Использовать бесплатные лимиты мобильного Naver Map SDK для отображения маршрутной нити и стрелок поворотов.

## Алгоритмы и UX-механики:
1. **Динамический масштаб**: Карта автоматически отдаляется на длинных прямых участках и плавно приближается (зумится) при подходе к перекресткам, развилкам или остановкам транспорта.
2. **Типы транспорта**: Поддержка трех профилей (Пешеход, Двухколесный транспорт/Самокат, Общественный транспорт с таймером прибытия).
3. **Эффект присутствия**: Во время ведения по маршруту 3D-модель Виолетты полностью скрывается, уступая место виджету. Связь с пользователем поддерживается через микро-звуки (вздохи/покашливания при пересчете пути) и короткие голосовые подсказки-ориентиры.
