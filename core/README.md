# СИСТЕМНОЕ ЯДРО И ПОДКЛЮЧЕНИЕ К ИИ (CORE & LLM SPECIFICATION)

## Ключевая задача:
Организовать стабильное, бесперебойное и полностью бесплатное подключение приложения к языковой модели Google Gemini для обеспечения работы "мозга" Виолетты.

## Архитектура Бесплатного Пула (Gemini API Free Tier):
1. Модель по умолчанию: **Gemini 1.5 Flash** (высокая скорость, поддержка текста, аудио и изображений, лимит 15 запросов в минуту на один ключ).
2. В коде приложения инициализируется массив (список) из нескольких бесплатных API-ключей: `const geminiKeys = [KEY_1, KEY_2, KEY_3];`.
3. **Алгоритм Round-Robin**: При каждом новом запросе пользователя приложение берет следующий ключ из списка. Если один ключ исчерпал минутный лимит (ошибка 429), система мгновенно и незаметно для пользователя переключается на рабочий ключ из пула.

## Логика работы с камерой (Мультимодальность):
1. Запрос "Виолетта, посмотри на это" активирует камеру.
2. Система опрашивает оборудование:
   - Если доступна USB-камера очков (`is_glasses_camera_available == true`), снимок делается с нее.
   - Если камеры в очках нет, Виолетта просит: *"Наведи на этот объект камеру смартфона, пожалуйста"*, и снимок делается с основной камеры телефона.
3. Картинка сжимается до разрешения 1024x1024 (для экономии трафика и батареи) и отправляется в Gemini API вместе с голосовым запросом пользователя.
