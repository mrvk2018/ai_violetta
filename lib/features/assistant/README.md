# ЯДРО АССИСТЕНТА ВИОЛЕТТЫ (ASSISTANT CORE FEATURE)

## Ключевая задача:
Обеспечить постоянное энергосберегающее фоновое присутствие ассистента Виолетты и управление её визуальным и звуковым поведением в AR-очках.

## Инженерные требования (Закон 0% нагрева и 0 вон):
1. **Фоновый сон (Wake-word)**: Использовать низкоуровневый `Android Background Service` (на iOS — режим работы при активном приложении с погашенным экраном). Микрофон циклично пишет звук в легкий буфер ОЗУ (1.5-2 сек). Расход батареи стремится к нулю.
2. **Пробуждение**: Мгновенный запуск интерфейса очков при локальном распознавании триггера "Эй, Виолетта" или "Хэй, Виолетта".
3. **Визуал (3D Low-Poly)**: Модель в полный рост в стиле Cel-shading (3D-аниме). Волосы полностью убраны под вязаную молодежную шапку-бини для экономии 40% ресурсов GPU при стереоскопическом рендеринге в очках.
4. **Анимация (Йода-стайл)**: Использовать ограниченный набор фиксированных зацикленных пресетов движения (`Idle`, `Speaking`, `Thinking`, `Happy`, `Surprise`). Персонаж зафиксирован в одной точке обзора в очках, отключая тяжелые алгоритмы сканирования пространства.

## Логика скрытия и психология присутствия:
1. При уходе в тень (переключение на навигатор или режим сна) модель плавно исчезает (`Fade out`), а в углу экрана загорается крошечный пульсирующий неоновый кружок (индикатор стабильной работы).
2. Перед скрытием Виолетта ОБЯЗАТЕЛЬНО запускает функцию `playRandomGoodbye()`, случайно выбирая одну из двух коротких фраз:
   - *"Я рядышком, на связи! Зови, если что"*
   - *"Я тут, в ушке. Позови — и я сразу появлюсь"*
3. В фоновом режиме навигации в аудиоканал раз в несколько минут могут подаваться тихие естественные триггеры присутствия (вздохи, легкое покашливание).
